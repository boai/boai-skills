# 写公众号影评 — 博爱光影

为微信公众号"博爱光影"撰写深度影评文章。核心原则：**每篇必须有独特分析角度，不做剧情复述机器；观点必有具体场景/数据/事实支撑。**

## 执行流程概览

| 阶段 | 步骤 | 产出 |
|------|------|------|
| **1. 选题确认** | 明确影片 → 查国家/类型 → 确定分类路径 → 确定独特切入点 | 分类路径 + 一句话角度 |
| **2. 调研** | WebSearch 搜公共事实（剧情/主创/票房/评分/幕后） | 调研笔记（只记事实，不记他人观点） |
| **3. 撰写** | 按文章结构模板写作，图片用标准 Markdown 语法嵌入 | 完整 .md 文章 |
| **4. 配图** | TMDb 下载海报+剧照 → 重命名 → PIL 生成封面头图 | 素材文件夹 |
| **5. 检查** | 对照检查清单逐项核对 → 清理临时脚本 | 发布就绪 |

1. 明确影片名称 → 确定制片国家 → 匹配分类路径

**规则**：

- 禁止打开完整影评"参考写法"

## 阶段 2：调研

### 必须搜索的内容

- 剧情事实、主创采访、票房数据、评分数据
- **幕后制作故事** — 拍摄花絮、换角风波、剧本改动、导演创作动机。这些事实能极大增强文章的深度和可读性
- 不同立场的影评观点 — 仅了解讨论语境，论述必须自己写

### 规则

- 调研笔记只记录事实和数据
- 禁止复制粘贴成段文字
- 禁止打开完整影评"参考写法"
- **🚨 搜索时必须带上完整片名+年份**，防止搜到同名片/翻拍版/剧集版的信息
- **🚨 事实必须可交叉验证**：关键数据（票房、评分、上映日期、导演、主演）至少从两个独立来源确认
- **🚨 记录来源 URL**：每条关键事实标注出处链接，方便后续核查

### 🚨 上映时间专项验证（最容易出错，必须严格）

上映时间是文章中最基本的事实，**写错就是硬伤，读者一眼看出不专业**。以下规则必须逐条执行：

**1. 区分日期类型，明确标注**
- 🎬 **首映/电影节首映**：最早公开亮相（威尼斯/戛纳/多伦多等）
- 🏠 **本土公映**：制片国家/地区的大规模影院上映日期
- 🇨🇳 **中国内地上映**：在中国大陆影院公映的日期（文章面向中国读者，这是最重要的日期）
- 📀 **流媒体上线**：Netflix/Disney+ 等平台上线日期（不是影院上映）

**2. 交叉验证流程**
```
搜索 1：Wikipedia "片名 (film)" → 查 Release 章节的 Theatrical 日期
搜索 2：豆瓣 "片名" → 查"上映日期"栏目
搜索 3：猫眼/IMDb "片名" → 查 Release info
```
**三个来源的日期必须一致，不一致就继续查，绝不猜。**

**3. 常见错误陷阱**
- ❌ 把电影节首映日期当公映日期写
- ❌ 把美国上映日期当中国上映日期写（或反过来）
- ❌ 把重映日期当首映日期写
- ❌ 把流媒体上线日期当影院上映日期写
- ❌ 把 IMDb 上粉丝填的日期当官方日期
- ❌ 翻拍版/续集的上映日期写到原版上

**4. 记录格式**（写入调研笔记）
```
《片名》上映时间：
- 首映：20XX年X月X日 @ XX电影节（来源：Wikipedia）
- 本土公映：20XX年X月X日（美国）（来源：IMDb）
- 中国内地上映：20XX年X月X日（来源：豆瓣）✅ 文章中使用此日期
```

## 阶段 3：文章撰写

### 文章结构模板

```markdown
> **↓↓↓ 点击下方卡片关注博爱光影 ↓↓↓**

---

# 文章主标题

> 引语/核心论点一句话

![封面头图](./素材文件夹/封面-头条.png)

---

## 正文各章节 ...

![剧照](./素材文件夹/剧照-01.jpg)

分析内容...

---

*互动问题，引导评论区留言。*

*关注我，系列持续更新中。下一期：《下期影片》—— 一句话预告。*

> **↓↓↓ 点击下方卡片关注博爱光影 ↓↓↓**

*本文写于YYYY年M月D日。*
*配图来源：剧照/海报来自TMDb官方公开素材。*
```

**关键差异**（与旧版模板对比）：
- 🚨 顶部和底部**不再使用扫码关注图片**，改用文字引导 `> **↓↓↓ 点击下方卡片关注博爱光影 ↓↓↓**`
- 尾部必须有**互动问题**和**下期预告**
- 封面头图 alt 用 `![XX封面]`，海报 alt 用 `![XX海报]`，剧照 alt 统一用 `![剧照]`（不写长描述）

### 关键规则

| 规则 | 说明 |
|------|------|
| 图片语法 | **标准 Markdown**：`![描述](相对路径)` |
| 相对路径 | 剧照/海报用 `./素材文件夹/` |
| 顶部引导 | 🚨 用文字 `> **↓↓↓ 点击下方卡片关注博爱光影 ↓↓↓**`，不用扫码图片 |
| 底部引导 | 🚨 同上，文字引导，不用扫码图片 |
| 封面 alt | `![XX封面]` 简洁格式，如 `![少林足球封面]` |
| 海报 alt | `![XX海报]` 简洁格式，如 `![少林足球海报]` |
| 剧照 alt | 🚨 统一用 `![剧照]`，不写长描述（因为无法确认图片实际内容与描述一致） |
| 不强制品牌图 | 不再在文章顶部放"博爱光影"品牌介绍图（不好看） |
| 配图数量 | 4-6 张剧照，多余的删掉（跟海报重复的、没用的都删） |
| 字数 | 3000–5000 字 |
| 段落 | 短段落，手机 4-6 行一段 |
| 尾部 | 必须包含互动问题 + 下期预告 + 日期声明 + 配图来源声明 |

### 标题策略

| 思路 | 说明 | 示例句式 |
|------|------|---------|
| **悬念反差** | 反常识判断制造好奇 | "都说X是Y，我看分明是Z" |
| **数字锚定** | 具体数量/时间跨度 | "三刷之后才看懂" |
| **细节切入** | 被忽视的场景/角色/道具 | "那个只有3秒的镜头" |
| **时代关联** | 电影与当下社会情绪 | "这部电影正在我们身边发生" |
| **类型颠覆** | 从类型规则入手 | "这部XX片，根本不是XX片" |

### 写作原则

1. **搜事实，不搜观点**
2. **证据导向** — 每个论点必有具体场景/镜头/台词/数据支撑
3. **独特切入点** — 不做"剧情+感想"流水账
4. **叙事不超过全文 30%** — 只在分析需要的范围内叙述剧情
5. **每 500-800 字抛出一个新信息** — 读者不知道的事实、数据或分析角度
6. **有活人感** — 用自己说话的方式写，允许口语节奏和个人判断

### 禁止事项

- ❌ 复制粘贴成段文字
- ❌ 剧情从头到尾复述
- ❌ "好看推荐"式水文
- ❌ 不标注出处的引用
- ❌ 空话形容词堆砌
- ❌ **描述不是本文所写电影的剧情或场景** — 写A片绝不说B片的内容
- ❌ **正文提到的场景/角色名与配图对不上** — 写"阿甘跑过纪念碑"就不能配《幸福终点站》的图
- ❌ **用"某部电影"的模糊描述凑字数** — 所有论据必须有具体片名、场景、时间点
- ❌ **上映时间写错** — 上映时间是文章硬事实，写错=整篇可信度崩塌。不记得就去查，不确认就不写
- ❌ **凭记忆写数字** — 票房、评分、成本、上映日期，任何一个数字都必须查实，不能"大概""好像""应该"

### 正文与配图的对应关系（🚨 配图不能是装饰，必须对得上）

**核心规则：先定场景需求，再找对应剧照。找不到对的就标记出来让用户确认。**

**执行流程**：
1. 文章正文写完定稿
2. 标记需要配图的位置（每 500-800 字一个配图点）
3. 为每个配图点写一句话描述："这张图应该展示 XX 场景 / XX 角色在 XX 情境"
4. 拿这句描述去 TMDb backdrops 里找对应的剧照
5. 找到的剧照内容与描述对得上 → 下载插入；对不上 → 换一张

**🚨 关键限制：Claude 无法目视图片内容**

TMDb 下载的图片文件**没有场景描述元数据**，Claude 的 Read 工具也无法显示图片。因此：
- ❌ **禁止用颜色/亮度/指纹算法判断图片内容** — 算法区分不了"同一场景不同角度"和"不同场景"，已被证实不可靠
- ❌ **禁止编造 alt 文本描述图片场景** — 写"师兄弟集结"但实际图上是别的场景，读者一眼看穿
- ✅ **必须请用户确认**：下载图片后，请用户快速过一眼，告诉每张图是什么场景，然后按用户说的场景把图放到对应的内容段落旁边
- ✅ **如果用户说某张图"没用""像网图""跟XX重复"，立即删除或替换**，不辩解

**配图确认流程**（下载完成后）：
```
1. 把每张剧照的编号列出来
2. 请用户一句话描述每张图的内容："剧照-01 是 XX 场景，剧照-02 是 YY 角色..."
3. 按用户描述重新排位：把对应场景的图放到对应段落下
4. 用户说重复的、没用的 → 删除
5. 用户说像网图的 → 从 TMDb 换一张
```

**对应规则**：

| 规则 | 说明 |
|------|------|
| **场景级对应** | 正文讨论"阿甘在长椅上讲人生"→ 配图必须就是那张长椅上的剧照，不能是阿甘跑步的图 |
| **角色级对应** | 正文分析"尹天仇在海边教柳飘飘演戏"→ 配图必须是两人在海边的场景，不能是尹天仇一个人在片场的图 |
| **情绪级对应** | 正文讲悲剧转折 → 配图选低沉/灰暗调的剧照；正文讲喜剧高光 → 配图选明亮/欢快的剧照 |
| **禁止万能配图** | 不能选一张"主演的大头照"到处插，每张图必须对应当前段落讨论的具体内容 |

**写完后的对应检查**（逐张过，用户辅助）：
```
□ 剧照-01 对应文章第 X 段，讨论的是 ____，用户说图里是 ____ → ✅/❌
□ 剧照-02 对应文章第 Y 段，讨论的是 ____，用户说图里是 ____ → ✅/❌
...（每张都写出对应关系，❌ 的就换图或调位置）
```

## 阶段 4：配图

### 🚨 配图红线（禁止事项，违反即返工）

> **以下行为绝对禁止，没有例外：**

| 红线 | 说明 |
|------|------|
| ❌ **使用其他电影的图片** | 哪怕画风相似、同导演、同类型，也绝不能用。配A片只能用A片的图 |
| ❌ **使用 AI 生成的电影剧照** | 禁止用任何 AI 工具生成"像某部电影"的图片充当剧照 |
| ❌ **使用记忆中的图片** | 禁止凭印象描述图片然后搜索下载，必须通过 TMDb ID 精确获取 |
| ❌ **图片与文章内容张冠李戴** | 每张图必须来自本文所写的电影，场景和角色必须对得上 |
| ❌ **封面头图用其他电影海报拼贴** | 封面头图中的电影元素（海报局部、角色剪影等）必须来自本片 |
| ❌ **使用重复/高度相似的图片** | 同一场景的多张剧照只选 1 张；海报只选 1 张主海报；禁止用不同编号引用同一张或高度相似的图 |
| ❌ **配图与上下文内容不相关** | 正文讨论场景A就不能配场景B的图，哪怕都是同一部电影。每张图必须对应当前段落的具体内容 |
| ❌ **海报用英文版/网图** | 华语电影必须用中文官方海报。海报要像正式公映海报（有片名、有演职员表排版），不能是粉丝自制图或视频截帧 |
| ❌ **用颜色算法代替目视判断** | 颜色指纹/亮度/方差等算法无法区分"同一场景不同角度"和"不同场景"。不靠算法断定图片内容或多样性 |

### 配图优先级

> **有真实剧照必须用真实的，搜不到才自制。封面头图不能当正文配图。**

| 优先级 | 类型 | 来源 | 命名 |
|:---:|------|------|------|
| **1** | 电影官方海报 | **TMDb** 搜 poster → 下载 → 验证 | `海报-官方.jpg` |
| **2** | 电影剧照 | **TMDb** 搜 backdrops → 下载 → 验证 | `剧照-01.jpg` ~ `剧照-NN.jpg` |
| **3** | 数据图表 | Python PIL 自制 | `配图-数据一览.png` |
| **4** | 封面头图 | Python PIL 自制（每部电影独立配色） | `封面-头条.png` |

### TMDb 图片下载流程（含强制验证）

#### 第一步：精确搜索影片

```
WebSearch: "site:themoviedb.org 《完整片名》 上映年份"
```

**关键**：必须匹配**影片名称 + 上映年份 + 导演**三项，三者都吻合才算找对。重名影片、翻拍版、同名剧集都可能在搜索结果中，必须仔细甄别。

如果搜索结果不唯一，用以下方式二次确认：
```
WebSearch: "site:themoviedb.org 《完整片名》 director 上映年份"
```

#### 第二步：验证 TMDb ID

确认找到的 TMDb 页面上影片名称、年份与要写的影片一致后，记录 TMDb ID。

**验证清单**（逐项确认后才能进入下载）：
- □ 片名完全匹配（原文片名和中文译名都对）
- □ 上映年份匹配
- □ 导演匹配
- □ 确认这不是同名翻拍/重制/剧集版

#### 第三步：筛选并下载图片

**海报选择（只选 1 张）**：
- TMDb 上通常有多个语言版本的海报变体（中文版、英文版、预告海报、角色海报等）
- **只下载 1 张**主海报，选择规则：

| 影片国家 | 海报优先级 |
|----------|-----------|
| **中国/华语电影** | 🚨 **必须中文版海报**。TMDb 上找 `zh` / `cn` 语言标签的海报，找不到中文版就继续搜，不用英文版凑合 |
| 日本电影 | 优先日文原版海报 |
| 韩国电影 | 优先韩文原版海报 |
| 美国/欧洲电影 | 优先英文原版海报，有中国内地引进版海报也可选 |

- 同一语言版本有多个变体（预告海报 vs 正式海报）→ 选正式公映海报，画面信息最完整的
- 命名：`海报-官方.jpg`

**剧照选择（场景必须分散，第一张必须最正）**：

🚨 **剧照-01 硬性要求**：
- 必须是**官方发布的原始剧照**（original still），不是粉丝截图、不是视频截帧
- **中国/华语电影 → 剧照-01 必须是中文语境下的官方剧照**（场景中有中文标识/中文背景/中文角色名）
- 这张图是文章第一张正文配图，代表整篇文章的视觉门面
- 优选：电影官方宣传照、主视觉剧照、最具代表性的场景照

**后续剧照筛选原则**：
- TMDb backdrops 通常有大量剧照，需人工筛选
- 每张剧照必须是**不同的场景/不同的角色组合/不同的情绪时刻**
- 同一场景的多个角度/连续帧 → 只选其中构图最好的一张
- 同一角色的多张特写 → 最多选 1 张，其他选群像或不同角色
- 最终选 4-8 张，覆盖影片**开头/中段/高潮/结尾**不同阶段

**筛选自检**（选完后对着列表过一遍）：
- □ 没有两张图来自同一个场景
- □ 海报只选了 1 张
- □ 中国/华语电影 → 海报是中文版 ✅
- □ 剧照-01 是官方原始剧照，不是截帧/粉丝图
- □ 中国/华语电影 → 剧照-01 是中文语境 ✅
- □ 剧照覆盖了影片不同段落
- □ 没有两张图是同一角色相同造型的特写

1. 访问 `themoviedb.org/movie/{id}/images/posters` → 逐个检查 `iso_639_1` 语言标签 → 按国家规则选 1 张
2. 访问 `themoviedb.org/movie/{id}/images/backdrops` → 按上述规则筛选 4-8 张，剧照-01 必须满足硬性要求
3. 用 `curl -L -o` 从 `image.tmdb.org/t/p/original/{hash}.jpg` 下载
4. 海报命名为 `海报-官方.jpg`，剧照按场景出现顺序命名为 `剧照-01.jpg` ~ `剧照-NN.jpg`

**每下载一张图，立即看一眼图片内容，确认**：
- □ 图片中的角色/场景属于**本文所写的电影**，不是其他电影
- □ 不是演员在其他作品中的剧照
- □ 海报上的片名就是本文所写的电影
- □ 🚨 **与已下载的图不重复**：和素材文件夹里现有的每张图对比，不是同一场景的不同角度/不同裁切版本

**如果发现图片不对或重复**：删除该文件，回到第三步换一张。

### 文章中图片引用规则

1. **alt 文本简洁统一**：
   - 封面：`![XX封面]`（如 `![少林足球封面]`）
   - 海报：`![XX海报]`（如 `![少林足球海报]`）
   - 剧照：**统一用 `![剧照]`**，不写场景描述。因为 Claude 无法目视确认图片实际内容，写描述反而容易写错（比如写"师兄弟集结"实际图上不是）
2. **正文引用编号必须与文件名一致**：文中写 `剧照-03.jpg`，文件就必须叫 `剧照-03.jpg`
3. **禁止用封面头图当正文配图**：封面头图只能出现在文章开头一次
4. **每张配图必须出现在相关的正文段落附近**，不能随意堆放
5. **多余的剧照果断删除**：跟海报内容重复的、看不出有用处的图，删掉不要凑数

### 封面头图设计规范

**不搞统一深蓝**。每部电影根据其视觉风格设计独立配色：

| 要素 | 规范 |
|------|------|
| 尺寸 | 900×383 |
| 配色 | 从电影海报/主题中提取，每部不同 |
| 标题字体 | 楷体/苹方 78pt+，必须大 |
| 底部字体 | 22pt+，必须有半透明深色底条确保可读 |
| 内容 | 电影名 + 一句金句/台词 + "星爷电影系列"标签 + "博爱光影"品牌 |
| 视觉元素 | 径向光影、胶片齿孔、强调线、装饰三角等（可选） |

**配色示例**：
- 功夫 → 深红棕底色 + 橙红标题
- 少林足球 → 深绿底色 + 金黄标题
- 喜剧之王 → 深海蓝底色 + 暖橙标题
- 食神 → 深棕底色 + 金黄标题
